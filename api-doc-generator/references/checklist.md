# 接口文档自查清单

> 完成文档撰写后、写入 Wiki 前必须逐项确认。

## 结构完整性

- [ ] 文档包含所有必需章节：标题、接口概述、基础信息表、入参说明表、嵌套入参说明表（如有）、返参说明表、场景/分支行为表（如有）、入参示例、返参示例、异常/边界行为表、错误码表、代码位置
- [ ] 章节顺序与模板一致
- [ ] 草稿已保存到临时目录（如 /tmp/api-doc.md）

## 内容质量

- [ ] 入参和返参字段使用表格格式
- [ ] 入参表包含：字段、位置、类型、必填、说明
- [ ] 返参表包含：字段、类型、说明
- [ ] 嵌套字段使用点号/数组下标记法（如 `searchParams.ext.pageId`、`data.promotions[].actCode`）
- [ ] 条件必填字段标注"条件必填"并说明具体条件
- [ ] 枚举值用独立表格列全
- [ ] 示例 JSON 语法正确，匹配 DTO 结构
- [ ] 异常/边界行为基于代码真实实现（返回 null、默认值、字段优先级）

## 源码对齐

- [ ] 已阅读 Controller、DTO、Service、Client 相关源码
- [ ] 字段说明基于源码，不是仅凭字段名猜测
- [ ] 外部 DTO 只记录本地代码/测试能证明的字段，或说明遵循外部契约
- [ ] 代码位置信息完整（Controller 类名、方法名、文件路径）

## 安全检查

- [ ] **未出现密钥、真实 token、完整 Cookie、密码**
- [ ] **未暴露数据库表名、SQL、Redis Key、缓存策略**
- [ ] **未包含 Java 代码示例**（`@Resource`、`import`、SDK 调用）
- [ ] **未在对外文档中出现内部类名**（`*VO` / `*DTO` / `*Client` / `*Service`）
- [ ] **未包含"快速开始"、"FAQ"等额外章节**

## Wiki 写入前检查

- [ ] Wiki 认证状态已确认（`node ~/.wiki-17u/scripts/wiki.mjs doc-status`）
- [ ] Wiki URL 包含有效的 fid 参数
- [ ] Markdown 内容已通过语法检查

## Wiki 写入后验证

- [ ] 执行验证命令：`node ~/.wiki-17u/scripts/wiki.mjs fetch-doc '<url>' --outputDir=/tmp/wiki-verify --meta --no-images`
- [ ] 表格仍然是表格（未被转义或丢失格式）
- [ ] 代码块仍然是代码块（未被转义或丢失格式）
- [ ] 章节结构完整
