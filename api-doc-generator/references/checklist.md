# 接口文档自查清单

> 完成文档撰写后、提交前必须逐项确认。

## 结构与存放

- [ ] 文件路径为 `docs/api/<业务名>.md`，业务化命名，不使用内部 Java 类名
- [ ] 文件编码为 **UTF-8（无 BOM）**，未出现乱码
- [ ] 文档**严格包含 5 章节**：接口描述、接口路径、请求与响应参数、错误码、业务流程图
- [ ] 章节顺序与模板一致，未额外增加章节

## 接口描述

- [ ] 不超过 5 行
- [ ] 用业务语言，无 Controller / Service / Client / SDK 等内部概念
- [ ] 包含适用场景

## 接口路径

- [ ] 列出 4 个环境完整 URL：qa / uat / pre / prd
- [ ] 明确标注 Method（GET/POST）、Content-Type、字符编码
- [ ] URL 已替换 `{path}` 为真实路径

## 请求 / 响应参数

- [ ] 请求参数表格含 `位置（Path/Query/Body）` 列
- [ ] 必填性、类型、说明完整
- [ ] 枚举值用独立表格列全，禁止只列"常用值"
- [ ] 响应外层 + `data` 内层分开列
- [ ] 提供成功 + 失败 JSON 响应示例

## 错误码

- [ ] 列出通用错误码（`0000` / `1000` / `2000` / `9999`）
- [ ] 列出本接口额外触发的业务错误码（如有）
- [ ] HTTP 层异常单独列表
- [ ] 每个错误码含"触发条件 + 调用方应对"

## 业务流程图

- [ ] 使用 excalidraw 绘制，源文件存于 `docs/api/excalidraw/<doc-name>-flow.excalidraw`
- [ ] 导出 PNG 存于 `docs/api/images/<doc-name>-flow.png`
- [ ] 文档中正确引用图片
- [ ] 图中**未暴露**数据库、缓存、Redis、内部服务等实现细节
- [ ] 流程节点用业务语言，仅体现"调用方 -> 接口 -> 关键判断 -> 返回"

## 禁止事项确认

- [ ] **未出现 Java 代码示例**（`@Resource`、`import`、SDK 调用）
- [ ] **未出现内部类名**（`*VO` / `*DTO` / `*Client` / `*Service`）
- [ ] **未暴露数据库表名、SQL、Redis Key、缓存策略**
- [ ] **未包含"快速开始"、"特殊行为"、"FAQ"等额外章节**
- [ ] 未使用 emoji
